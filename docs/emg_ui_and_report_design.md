# 肌电展示界面与报告分析说明

本文说明当前界面中肌电信息的展示意义、弹出曲线窗口的作用，以及报告中肌电指标的计算逻辑。

## 1. 主界面肌电区的意义

主界面右侧的“肌电监测”是训练过程中的快速观察区，目标是让治疗师在不离开训练页的情况下判断：

- 肌电链路是否接入真实设备；
- CH1 / CH2 当前是否有有效发力；
- 两个通道的发力强度是否明显不平衡；
- 是否出现疲劳或异常波动的趋势。

主界面不适合放大量曲线，因此只保留状态、通道强度条、疲劳指数和一条轻量级小波形。

## 2. 右下角小波形的意义

主界面中的小波形展示的是最近一段时间内 CH1 / CH2 的肌电强度变化：

- 蓝色：CH1；
- 绿色：CH2；
- 指标：肌电强度（RMS）。

它的用途是“实时粗看”：治疗师可以快速判断患者是否在持续发力、发力是否突然中断、两侧或两块肌肉是否差异明显。

小波形不是完整报告图，也不用于自动暂停。它只显示上位机实际收到的肌电特征数据，不生成模拟数据。

## 3. 弹出肌电曲线窗口的意义

点击“查看肌电曲线”后，会弹出更大的肌电特征窗口。这个窗口左右两侧分别展示 CH1 和 CH2，每个通道按层展示多个特征趋势。

弹窗的意义是把“实时监测”和“详细分析”分开：

- 主界面保持清爽，不被复杂曲线挤满；
- 治疗师需要细看时，可以在弹窗中观察每个通道的特征变化；
- 弹窗顶部显示当前接收采样率/特征帧率，便于判断数据链路是否稳定；
- 弹窗展示的是实时接收到的特征帧，不会用模拟肌电补数据。

## 4. 肌电指标中文解释与计算方法

### 肌电强度（RMS）

RMS 表示一段时间窗内肌电信号的均方根值，用来反映肌肉收缩强度。

计算方式：

`RMS = sqrt(mean(x²))`

数值越高，通常表示该时间窗内肌肉激活越强。

报告中的原始肌电窗口特征会先对每个窗口做去均值处理，用来降低直流偏置或基线漂移的影响。

### 平均绝对肌电值（MAV）

MAV 表示一段时间窗内肌电信号绝对值的平均水平，用来观察整体发力强弱。

计算方式：

`MAV = mean(abs(x - mean(x)))`

### 积分肌电值（IEMG）

IEMG 表示一段时间窗内肌电活动的累计量，适合观察某个动作周期中的总肌肉参与程度。

计算方式：

`IEMG = sum(abs(x - mean(x)))`

### 波形长度（WL）

WL 表示相邻采样点变化量的累计长度，用来观察肌电信号复杂度和发力变化程度。

计算方式：

`WL = sum(abs(x_centered[i] - x_centered[i-1]))`

### 过零次数 / 过零率（ZC / ZCR）

过零次数表示信号穿越零点的次数，过零率表示单位时间或单位窗口内过零的比例。它可以辅助观察肌肉激活切换是否频繁，也可反映一定程度的信号振荡特征。

计算方式：

`ZC = count(x_centered[i-1] * x_centered[i] < 0)`

### 波动系数（CV）

CV 用于描述信号在窗口内的相对波动程度，可辅助观察发力是否稳定。

常见计算方式：

`CV = std(x) / mean(abs(x))`

### 疲劳指数

疲劳指数由下位机/肌电处理链路输出，用于提示肌肉疲劳倾向。当前界面和报告仅将其作为训练观察指标，不作为临床诊断结论。

## 5. 报告中的肌电分析定位

报告中的肌电分析用于回答三个问题：

1. 患者有没有主动发力？
2. 发力是否稳定？
3. 训练过程中是否出现疲劳或震颤/不稳定趋势？

动作报告更适合放完整曲线；总报告更适合展示多个动作的肌电摘要和趋势入口。这样既能保留详细分析，又不会让总报告过于臃肿。
